# Shared UI Components (single source of truth)

Every UI building block in this project lives in
`apps/frontend/src/components/shared`. This is the **only** place UI components
may come from.

## Hard rules

1. **Always import from shared.** Use the `@/components/shared/...` alias.
   Never import a UI component from a third-party path directly in a page (wrap
   it in shared instead), and never write a raw inline `<button>`, `<input>`,
   `<select>`, table, spinner, modal, etc. when a shared component exists.
2. **If a component is missing, create it in shared FIRST.** Do not improvise it
   inline in the page. Add a new file under `src/components/shared` (or the
   right subfolder), export it, then import it where needed.
3. **Match existing conventions** when creating a new shared component:
   - `"use client"` at the top if it uses state/effects/handlers.
   - Tailwind utility classes for styling (see `CustomButton.tsx`).
   - Root-level components (`CustomButton`, `Loader`) use **default exports**.
   - `form/` and `table/` components use **named exports** re-exported from an
     `index.ts` barrel — add new ones to that barrel too.
   - Keep internal imports relative within a folder so the folder stays portable
     (the `form/` folder is copy-paste portable — see `form/README.md`).
4. **Never bypass shared** by copying component code into a page or another
   non-shared location.

## What already exists in shared

### Root — `@/components/shared/...`

- `CustomButton` (default export) — variants: `primary | secondary | danger | ghost | icon`, `iconColor: slate | blue | red`. Import: `import CustomButton from "@/components/shared/CustomButton"`.
- `Loader` (default export) — sizes: `sm | md | lg`. Import: `import Loader from "@/components/shared/Loader"`.

### Forms — `@/components/shared/form`

Form wrapper: `RHFZodForm` (react-hook-form + zod).
Plain (uncontrolled) fields: `PlainInput`, `PlainTextarea`, `PlainSelect`, `PlainDateInput`, `PlainDateTimeInput`, `PlainPhoneInput` (+ `PLAIN_PHONE_COUNTRY_OPTIONS`).
RHF fields: `FormInput`, `FormTextarea`, `FormSelect`, `FormMultiSelect`, `FormCheckbox`, `FormRadioGroup`, `FormDateInput`, `FormDateTimeInput`, `FormFileUpload`, `FormPhoneInput`, `FormPhoneCountryField`, `FormOtpInput`, `FormSwitch`, `FormSlider`, `FormTagsFieldArray`, `FormAutocomplete`, `FormSubmitButton`.
Advanced fields: `FormAddressAutocomplete`, `FormAdvancedFileUpload`, `FormAsyncSelect`, `FormColorInput`, `FormCreatableSelect`, `FormCurrencyInput`, `FormDateRangeInput`, `FormMapPicker`, `FormMaskedInput`, `FormPercentUnitInput`, `FormRatingInput`, `FormRecurringSchedule`, `FormRichText`, `FormSignaturePad`, `FormTreeSelect`, `FormUrlInput`.

Example: `import { RHFZodForm, FormInput, FormSubmitButton } from "@/components/shared/form";`

### Tables — `@/components/shared/table`

- `DataTable` — feature-rich TanStack table. `import { DataTable } from "@/components/shared/table";`

### shadcn/ui primitives — `@/components/shared/shadcn`

Low-level primitives generated by the shadcn CLI (style **new-york**, built on
`radix-ui` + Tailwind **v4**). Every component is re-exported from the folder's
`index.ts` barrel, so import by name:

`import { Button, Dialog, Card, Select, Tabs, Tooltip, Badge } from "@/components/shared/shadcn";`

Available (56): accordion, alert, alert-dialog, aspect-ratio, avatar, badge,
breadcrumb, button, button-group, calendar, card, carousel, chart, checkbox,
collapsible, combobox, command, context-menu, dialog, direction, drawer,
dropdown-menu, empty, field, form, hover-card, input, input-group, input-otp,
item, kbd, label, menubar, native-select, navigation-menu, pagination, popover,
progress, radio-group, resizable, scroll-area, select, separator, sheet,
sidebar, skeleton, slider, sonner, spinner, switch, table, tabs, textarea,
toggle, toggle-group, tooltip.

- Config lives in `apps/frontend/components.json` (`aliases.ui` → this folder).
  Add more with the shadcn CLI: `npx shadcn@latest add <name>` (requires
  **Node 20+** — use `nvm use 22`).
- The shadcn `form` primitive (`Form`, `FormField`, ...) is RHF wiring and is
  **distinct** from `@/components/shared/form` (the project's own RHFZodForm +
  `Form*`/`Plain*` fields). For app forms prefer `@/components/shared/form`; use
  the shadcn `form` primitive only when composing a shadcn-native form.

**Which to reach for:** app-level fields/forms/tables → `form` / `table` /
`CustomButton` / `Loader`. Generic primitives (dialogs, popovers, menus,
tooltips, etc.) → `shadcn`. Need something new? Compose it from shadcn
primitives inside a new `src/components/shared` file — never inline in a page.

## Workflow before writing any UI

1. Check the lists above. Does a suitable shared component exist? → import & use it.
2. Not listed? → re-scan `src/components/shared` (it may have grown). If still
   absent → **create it in shared**, export it (add to the relevant `index.ts`
   barrel if there is one), then import & use it.
3. Never leave UI building blocks defined inside a page or a non-shared file.
