## গল্প

ধরো তুমি ঢাকায় থাকো, আর তোমার দাদু থাকেন রংপুরের এক গ্রামে। দাদুকে একটা লম্বা চিঠি লিখলে — পুরো দশ পাতা! কিন্তু ডাকঘরের নিয়ম হলো, একটা খামে সর্বোচ্চ এক পাতা পাঠানো যাবে। তাহলে কী করবে? সহজ — দশটা খামে দশ পাতা ভরবে, প্রতিটা খামের উপরে লিখবে দাদুর ঠিকানা, তোমার নিজের ঠিকানা, আর কোণায় ছোট্ট করে নম্বর দেবে: "পাতা ১/১০", "পাতা ২/১০"… যাতে দাদু পরে সাজিয়ে পড়তে পারেন।

এবার মজার ব্যাপার — দশটা খাম যে একই পথে যাবে, তার কোনো গ্যারান্টি নেই। কোনোটা যাবে গাইবান্ধা হয়ে, কোনোটা বগুড়া হয়ে। মাঝপথে প্রতিটা ডাকঘরের কর্মী খামের ঠিকানা দেখে ঠিক করেন — "হুম, রংপুর যাবে? তাহলে এই গাড়িতে তোলো।" কেউ পুরো রাস্তা জানে না, সবাই শুধু জানে **পরের ধাপটা** কোথায়। শেষমেশ সব খাম দাদুর হাতে পৌঁছায়, দাদু নম্বর মিলিয়ে সাজিয়ে নেন, আর পুরো চিঠিটা পড়ে ফেলেন। 😊

বিশ্বাস করো আর না-ই করো — Internet ঠিক **এভাবেই** কাজ করে! তুমি যখন YouTube-এ ভিডিও দেখো বা WhatsApp-এ মেসেজ পাঠাও, পেছনে ঠিক এই ডাকঘরের খেলাটাই চলে, শুধু কোটি গুণ দ্রুত।

## Concept

**Internet মানে কী?** খুব সহজ করে বললে — Internet হলো পৃথিবীর সব computer-কে জুড়ে দেওয়া একটা বিশাল network of networks। তোমার বাসার Wi-Fi একটা ছোট network, তোমার university-র network আরেকটা, Google-এর data center আরেকটা — এই লাখ লাখ ছোট network যখন একে অপরের সাথে কথা বলতে পারে, সেটাই Internet।

এবার গল্পের চরিত্রগুলোকে technical নামে চিনে নিই:

| গল্পে         | Internet-এ                          | কাজ                            |
| ------------- | ----------------------------------- | ------------------------------ |
| দশ পাতার চিঠি | তোমার data (ছবি, মেসেজ, ভিডিও)      | যা পাঠাতে চাও                  |
| একেকটা খাম    | **Packet**                          | data-র ছোট ছোট টুকরা           |
| খামের ঠিকানা  | **IP address**                      | কোথা থেকে, কোথায় যাবে         |
| ডাকঘর         | **Router**                          | packet-কে সঠিক পথে forward করে |
| ডাক বিভাগ     | **ISP** (Internet Service Provider) | তোমাকে Internet-এ যুক্ত করে    |

**Packet কেন দরকার?** তোমার পাঠানো একটা ছবি হয়তো 3 MB। এত বড় জিনিস একসাথে পাঠালে network-এর তার "দখল" হয়ে থাকত, অন্য কেউ কিছু পাঠাতে পারত না। তাই data-কে ভাঙা হয় হাজার হাজার ছোট packet-এ (সাধারণত ~1500 bytes করে)। প্রতিটা packet-এ থাকে: কার কাছে যাবে (destination IP), কে পাঠালো (source IP), আর sequence number — ঠিক খামের গায়ের লেখাগুলোর মতো।

**Router কীভাবে কাজ করে?** Router হলো সেই ডাকঘরের কর্মী। সে packet-এর destination address দেখে আর নিজের routing table দেখে ঠিক করে — "এই packet পরের কোন router-এর কাছে পাঠাবো?" মজার ব্যাপার, কোনো router-ই পুরো পথ জানে না; প্রত্যেকে শুধু **next hop** ঠিক করে। তোমার packet ঢাকা থেকে America-র server-এ যেতে হয়তো ১৫-২০টা router পার হয় — একে বলে hops। আর একই message-এর দুটো packet ভিন্ন ভিন্ন পথে যেতেই পারে — কোনো সমস্যা নেই, গন্তব্যে গিয়ে আবার সাজানো হবে।

**ISP-র ভূমিকা:** তোমার বাসার broadband (যেমন Link3, Carnival) বা mobile data (Grameenphone, Robi) — এরাই তোমার ISP। এরা তোমার ছোট্ট home network-কে বড় Internet-এর সাথে জুড়ে দেয়। ছোট ISP-রা আবার বড় ISP-র সাথে যুক্ত, আর দেশের সব traffic শেষমেশ submarine cable (সমুদ্রের নিচের fiber optic তার!) দিয়ে বাকি পৃথিবীতে পৌঁছায়। হ্যাঁ, তোমার TikTok ভিডিওটা আক্ষরিক অর্থেই বঙ্গোপসাগরের তলা দিয়ে সাঁতরে আসে। 🌊

**Client আর Server:** এবার শেষ বড় ধারণা। Internet-এ দুই ধরনের চরিত্র আছে:

- **Client** — যে কিছু **চায়**। তোমার phone-এর browser যখন `facebook.com` খোলে, সে client।
- **Server** — যে **সেবা দেয়**। Facebook-এর data center-এ বসে থাকা শক্তিশালী computer-গুলো server; তারা ২৪ ঘণ্টা অপেক্ষা করে — কেউ request করলেই response পাঠায়।

```
[তোমার phone]  --request packet-->  [router] -> [router] -> ... -->  [Facebook server]
[তোমার phone]  <--response packets--  [router] <- [router] <- ... <--  [Facebook server]
```

পুরো ঘটনাটা ঘটে কয়েকশ millisecond-এ। ভাবো — তোমার request রংপুর না, একেবারে Singapore বা America ঘুরে আসে, চোখের পলকের চেয়েও কম সময়ে!

একটা ভুল ধারণা ভেঙে দিই: Internet কোনো একটা কোম্পানির জিনিস না, কোনো "central computer"-ও নেই যে সব চালায়। এটা decentralized — লাখো network স্বেচ্ছায় কিছু common নিয়ম (protocol) মেনে চলে বলেই সব কাজ করে। সামনের lesson-গুলোতে আমরা সেই নিয়মগুলোই শিখবো — DNS, TCP/IP, HTTP।

## Real World

**Facebook-এর মহাবিপর্যয় (October 2021):** একদিন হঠাৎ Facebook, WhatsApp, Instagram — সব একসাথে গায়েব! প্রায় ৬ ঘণ্টা পৃথিবীর কেউ ঢুকতে পারেনি। কারণ? Facebook ভুল করে নিজেদের router-গুলোর routing ঘোষণা (BGP route) Internet থেকে মুছে ফেলেছিল। ফলে পৃথিবীর কোনো router আর জানত না Facebook-এর server-এ পৌঁছানোর "পরের ধাপ" কোনটা — ঠিক যেন ডাকঘর থেকে রংপুরের ঠিকানাটাই মুছে গেছে। মজার (এবং করুণ) ব্যাপার: Facebook-এর engineer-রা নিজেদের office-এর দরজাও খুলতে পারছিলেন না, কারণ door system-ও ওই network-এর উপর নির্ভরশীল ছিল!

**Submarine cable কাটা পড়লে:** বাংলাদেশ মূলত SEA-ME-WE submarine cable-গুলোর উপর নির্ভরশীল। ২০২৪ সালে এরকম একটা cable-এ সমস্যা হওয়ায় সারা দেশের Internet ধীর হয়ে গিয়েছিল। কারণ packet-গুলোর প্রধান "highway" বন্ধ — বিকল্প পথে ঘুরে যেতে হচ্ছিল, তাই latency বেড়ে গিয়েছিল।

## মনে রাখো

- Internet হলো network of networks — কোনো central মালিক বা central computer নেই।
- Data ছোট ছোট **packet**-এ ভাগ হয়ে যায়; প্রতিটা packet-এ source IP, destination IP আর sequence number থাকে।
- **Router** পুরো পথ জানে না — শুধু next hop ঠিক করে; একই message-এর packet ভিন্ন পথে যেতে পারে।
- **ISP** তোমার device-কে Internet-এ যুক্ত করে; দেশের traffic submarine cable দিয়ে বিশ্বে পৌঁছায়।
- **Client** request পাঠায়, **server** response দেয় — এটাই Internet-এর মৌলিক কথোপকথন।
- Facebook-এর 2021 outage দেখিয়েছে — routing তথ্য হারালে server সচল থাকলেও কেউ তাকে খুঁজে পায় না।
