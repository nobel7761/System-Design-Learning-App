## গল্প

GolpoGram-এর প্রথম investor meeting। 😎 টেবিলের ওপাশে বিনিয়োগকারী ভদ্রলোক চশমার ফাঁক দিয়ে তাকিয়ে বললেন — "আপনি তো বলছেন আপনার platform-এ লাখো user আসবে। আচ্ছা বলুন তো, একজন user browser-এ `golpogram.com` লিখে Enter চাপার পর **ঠিক কী কী ঘটে**?"

এটা পৃথিবীর সবচেয়ে বিখ্যাত system design প্রশ্ন — কারণ এক প্রশ্নেই গোটা World 0 ঢুকে আছে। তুমি এক চুমুক পানি খেয়ে শুরু করলে — আর ৭টা lesson-এর সব টুকরো একে একে জোড়া লাগতে থাকলো।

## Full Journey — ধাপে ধাপে

**ধাপ ১ — URL ভাঙা (browser-এর মনে মনে):** Browser প্রথমে ঠিকানাটা পড়ে: scheme `https` (তালাওয়ালা আলাপ হবে), host `golpogram.com` (কার সাথে), port অলিখিত মানে `443`, path `/` (কোন পাতা)।

**ধাপ ২ — DNS: নাম থেকে নম্বর (lesson 2):** `golpogram.com`-এর IP কত? খোঁজ চলে সিঁড়ি বেয়ে — browser cache → OS cache → resolver (ISP/8.8.8.8) → দরকারে root → TLD (`.com`) → authoritative server। উত্তর এলো: `76.76.21.21`। Resolver টা cache-ও করে রাখলো (TTL পর্যন্ত) — পরের জনের জন্য।

**ধাপ ৩ — TCP: লাইন বসানো (lesson 3):** এবার ওই IP-র port 443-এ connection — তিন-পদের নাচ: `SYN` → `SYN-ACK` → `ACK`। লাইন তৈরি; এখন থেকে packet হারালে retry হবে, ক্রম ঠিক থাকবে — registered post service চালু।

**ধাপ ৪ — TLS handshake: তালা লাগানো (lesson 5):** Client Hello → Server Hello + **certificate** → browser যাচাই করলো CA-র সিলমোহর, নাম, মেয়াদ → key exchange-এ জন্ম নিলো **session key** → 🔒 এখন থেকে সব কথা encrypted। পথের কেউ দেখলে দেখবে শুধু আবর্জনা।

**ধাপ ৫ — HTTP request: অর্ডার যায় (lesson 4):** Encrypted টানেল দিয়ে গেলো অর্ডার-slip:

```
GET / HTTP/2
Host: golpogram.com
Cookie: token=eyJhbG...        ← পরিচিত user হলে (lesson 6!)
```

**ধাপ ৬ — Server-এর রান্নাঘর:** Request পৌঁছালো server-এ (সামনে হয়তো load balancer — World 1-এর trailer! 🎬)। Backend route মেলালো, cookie/JWT যাচাই করলো (গেটের ভাইয়া স্ট্যাম্প দেখলো), database/API থেকে data আনলো (lesson 7-এর চুক্তিমাফিক), বানালো response।

**ধাপ ৭ — Response ফিরে আসে:** `200 OK` + HTML। Browser পড়তে শুরু করলো, আর দেখলো — আরো জিনিস লাগবে: CSS, JavaScript, ছবি। প্রতিটার জন্য আবার request (ভাগ্যিস HTTP/2-তে এক connection-এই সব পাশাপাশি চলে!)। কিছু জিনিস browser cache-এ ছিল — `304 Not Modified`, পাঠাতেই হলো না।

**ধাপ ৮ — পর্দায় ছবি:** HTML → DOM, CSS → style, JavaScript চললো — পিক্সেল ফুটলো পর্দায়। Enter চাপা থেকে এই পর্যন্ত: **এক সেকেন্ডেরও কমে**, অথচ মাঝে ঘটে গেলো ডাকঘর, phonebook, handshake, নোটারি, restaurant — সাতটা গল্পের সবগুলো।

Investor সাহেব মুচকি হেসে বললেন — "Impressive। কাল টাকা পাঠাচ্ছি।" 💰

## Boss fight-এর আগে চেকলিস্ট

যাত্রার ক্রমটা চোখ বন্ধ করে বলতে পারো তো? এই ক্রম-প্রশ্ন boss-এ আসবেই:

1. **URL parse** — scheme/host/port/path আলাদা হয়
2. **DNS lookup** — cache সিঁড়ি বেয়ে নাম → IP (UDP, port 53)
3. **TCP 3-way handshake** — SYN → SYN-ACK → ACK (port 443-এ)
4. **TLS handshake** — certificate যাচাই → session key → 🔒
5. **HTTP request** — method + path + headers (+cookie/JWT)
6. **Server processing** — route → auth যাচাই → data → response তৈরি
7. **HTTP response** — status code + body; বাড়তি resource-এর জন্য আরো request, cache হলে 304
8. **Render** — HTML/CSS/JS থেকে পর্দায় ছবি

আর মনে রেখো প্রতিটা ধাপের _কেন_: DNS আগে কারণ IP ছাড়া TCP-র গন্তব্যই নেই; TCP আগে TLS-এর, কারণ লাইন ছাড়া তালা লাগাবে কীসে; TLS আগে HTTP-র, কারণ অর্ডার-slip খোলা পাঠানো চলবে না। শুভকামনা, যোদ্ধা! ⚔️
