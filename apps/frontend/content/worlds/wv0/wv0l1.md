## 🧭 CONCEPT JOURNEY

### Step 1: সবকিছু একটা উল্টো গাছ — root `/`

Instructor বোর্ডে প্রথমেই যেটা আঁকলেন — একটা গাছ, কিন্তু **উল্টো।** শিকড় (root) সবার উপরে, ডালপালা নিচে:

```
/                  ← root — বিল্ডিংয়ের মেইন গেট, সব কিছুর শুরু
└── home           ← সব ব্যবহারকারীর থাকার ফ্লোর
    └── code       ← তোমার নিজের ঘর
        └── lab
            └── projects
                └── websites
```

উপরের `/`-টার নাম **root**। Windows-এ যেমন `C:\`, Linux-এ তেমন `/` — কিন্তু D drive, E drive বলে কিছু নেই। **একটাই গাছ, সবকিছু ওই একটা `/` থেকে নেমেছে।**

> **Analogy:** `/` হলো বাংলাদেশের ম্যাপে "ঢাকা"। যেকোনো ঠিকানা শেষমেশ ঢাকা থেকেই শুরু — ঢাকা → মিরপুর → রোড ১০ → বাসা ৫। Linux-এ: `/` → `home` → `code` → `lab`।

শব্দ চিনে রাখো: **directory** = ফোল্ডার (একই জিনিস, Linux-এর ভাষায়), **file** = ভেতরের কাগজপত্র।

### Step 2: `pwd` — "আমি এখন কোথায়?"

```bash
pwd
```

```
/home/code
```

`pwd` = **p**resent **w**orking **d**irectory। উত্তর সবসময় root `/` থেকে পুরো রাস্তা — "মেইন গেট থেকে ঢুকে, home ফ্লোরে, code ঘরে।"

তোমার JS মাথার জন্য: `pwd` হলো টার্মিনালের `console.log(window.location.pathname)`।

### Step 3: `ls` — "এই ঘরে কী কী আছে?"

```bash
ls
```

```
lab  notes.txt  script.sh
```

`ls` = **l**i**s**t। আর এখানেই instructor-এর কনসেপ্ট ম্যাপ — তিনটা আলাদা প্রশ্ন, তিনটা আলাদা অস্ত্র:

| প্রশ্ন                              | অস্ত্র            |
| ----------------------------------- | ----------------- |
| এই ঘরে **কী কী আছে**?               | `ls`              |
| অমুক **ফাইলটা কোথায়**?             | `find` / `locate` |
| কোন ফাইলের **ভেতরে** অমুক লেখা আছে? | `grep`            |

এই তিনটার তফাতই আজকের লেকচারের মেরুদণ্ড।

### Step 4: `cd` — ঘর বদলানো, আর আজকের সবচেয়ে বড় কনসেপ্ট

ঘর বদলাতে `cd` (**c**hange **d**irectory)। কিন্তু path লেখার দুটো ভঙ্গি আছে, আর **এই তফাতটাই আজকের লেসনের হৃদয়:**

**Absolute path — পুরো ঠিকানা, `/` থেকে শুরু:**

```bash
cd /home/code/lab/projects
```

> চিঠির খামে পুরো ঠিকানা: "বাংলাদেশ, ঢাকা, মিরপুর, রোড ১০, বাসা ৫।" **যেখানেই দাঁড়িয়ে থাকো**, এই ঠিকানা একই জায়গায় পৌঁছাবে।

**Relative path — এখন যেখানে আছি, সেখান থেকে দিকনির্দেশ:**

```bash
cd lab/projects
```

> রিকশাওয়ালাকে বলা: "সামনে গিয়ে বাঁয়ে।" ফলাফল নির্ভর করে **তুমি এখন কোথায়** তার উপর — অন্য জায়গা থেকে একই কথায় অন্য গন্তব্য!

চেনার নিয়ম একটাই, instructor বোর্ডে দুবার আন্ডারলাইন করলেন: **শুরুতে `/` থাকলে absolute, না থাকলে relative।**

তিনটা শর্টকাট:

| চিহ্ন | মানে                     | মনে রাখার কায়দা                       |
| ----- | ------------------------ | -------------------------------------- |
| `~`   | তোমার home directory     | "নিজের বাড়ি" — `cd ~` মানে বাড়ি ফেরা |
| `.`   | এই ঘর (current)          | "এখানে"                                |
| `..`  | এক ধাপ উপরের ঘর (parent) | "বেরিয়ে পেছনে"                        |

### Step 5: `mkdir -p` — একসাথে পুরো গলি বানাও

নতুন directory বানাতে `mkdir`। Instructor সরাসরি দেখালেন আসল কায়দা:

```bash
mkdir -p ~/code/lab/projects/websites
```

`-p` ছাড়া চালালে Linux ঘ্যানঘ্যান করবে — "lab-ই তো নেই, projects বানাবো কীভাবে?" `-p` (**p**arents) বললে **মাঝের সব ধাপ নিজে বানিয়ে নেয়**, আর কোনোটা আগে থেকে **থাকলেও error দেয় না**।

> **Analogy:** `-p` সেই রাজমিস্ত্রি, যাকে "৫ তলায় ঘর বানাও" বললে দরকারমতো ১-২-৩-৪ তলাও বানিয়ে ফেলে; আর ৩ তলা আগে থেকে থাকলে "আছে তো, ভালো" বলে উপরে ওঠে।

### Step 6: `echo` আর `>` — কথা বলা, আর কথাটা ফাইলে ভরা

```bash
echo "Hello World"
```

```
Hello World
```

`echo` = JS-এর `console.log()`। কিন্তু instructor বোর্ডের কোণায় তিনটা শব্দ লিখলেন — **stdin, stdout, stderr।** প্রতিটা কমান্ডের তিনটা পাইপলাইন:

- **stdin** — কানে ঢোকা input
- **stdout** — মুখ দিয়ে বের হওয়া স্বাভাবিক output
- **stderr** — কান্নাকাটির আলাদা লাইন (error)

স্বাভাবিকভাবে stdout স্ক্রিনে যায়। `>` দিয়ে সেই স্রোত **ঘুরিয়ে ফাইলে ঢোকানো যায়:**

```bash
echo "Hello World" > ~/code/lab/projects/websites/index.html
```

স্ক্রিনে কিছু আসবে না — কথাটা ঢুকেছে `index.html`-এ। ফাইল ছিল না? `>` বানিয়ে নেয়। **এক লাইনে ফাইল তৈরি + লেখা** — VS Code-ও লাগলো না।

> **Analogy:** `echo` হলো কল, `>` হলো কলের মুখে পাইপ। পাইপ না লাগালে পানি মেঝেতে (স্ক্রিনে); লাগালে বালতিতে (ফাইলে)।

⚠️ একটা বিপদ আগেই জেনে রাখো: `>` ফাইলের **আগের সবকিছু মুছে** লেখে। আগেরটার নিচে যোগ করতে `>>`।

### Step 7: `find` — গোয়েন্দা লাগাও, নামে খোঁজো

লাখ লাখ ফাইলের সার্ভারে "ফাইলটা কোথায় রাখলাম?" — ঘরে ঘরে `ls` করা অসম্ভব। গোয়েন্দা ডাকো:

```bash
find ~/lab/projects -name "12-03-2026.log"
```

গঠনটা পড়ো: `find <কোথা থেকে খুঁজবে> -name "<কী নামের>"`। পুরো নাম মনে নেই? **wildcard** `*` = "এখানে যা খুশি":

```bash
find / -name "*.log"              # পুরো সার্ভারে সব .log
find / -name "*.log" -type f      # -type f = শুধু file
find / -name "log*"  -type d      # -type d = শুধু directory
```

Instructor সুন্দর একটা কথা ছুঁয়ে গেলেন — `find` ভেতরে কী করে? সে **উল্টো গাছটা ডাল ধরে ধরে হাঁটে** — এক ঘরে ঢোকে, তার ভেতরের ঘরে, তারপর তার ভেতরে — computer science-এ যার নাম **tree traversal (DFS)**। তোমার DSA course-এর tree chapter বাস্তবে এখানে!

আর `locate`? সে-ও খোঁজে, কিন্তু আগে থেকে বানানো **ইনডেক্স খাতা** দেখে — বিদ্যুৎগতির, কিন্তু খাতা পুরনো হলে নতুন ফাইল পায় না। `find` জ্যান্ত খোঁজে — ধীর, কিন্তু সবসময় সত্যি।

### Step 8: `grep` — ফাইলের ভেতরে খোঁজো

`find` খোঁজে **নাম**। প্রশ্ন যদি হয় — "**কোন ফাইলের ভেতরে** 'meeting' লেখা?" — তখন `grep`:

```bash
grep -r "meeting" /
```

- `-r` — recursive: ভেতরের সব ফোল্ডারের সব ফাইলে
- output ফরম্যাট: `ফাইলের-path:match-করা-লাইন`

তোমার চেনা জগতে `grep` = VS Code-এর **Ctrl+Shift+F** — শুধু পুরো সার্ভার জুড়ে।

> **এক লাইনে:** `find` পড়ে খামের উপরের **নাম**, `grep` খুলে পড়ে চিঠির **ভেতরটা**।

### Step 9: pipe `|` — এক কমান্ডের মুখ, আরেক কমান্ডের কান

নিয়ম এক বাক্যে: **বাঁ পাশের stdout, ডান পাশের stdin হয়ে যায়।**

```bash
ls /var/log | grep "nginx"
```

`ls`-এর তালিকা স্ক্রিনে না গিয়ে `grep`-এর কানে গেল → `grep` ছেঁকে শুধু "nginx" লাইনগুলো দেখালো।

> **Analogy:** রেস্টুরেন্টের লাইন — একজন কাটে, পরের জন ভাজে, পরের জন সাজায়। কেউ মেঝেতে ফেলে না, সোজা পরের জনের হাতে। `কাটো | ভাজো | সাজাও`।

## 🚫 COMMON MISTAKES — প্রথম সপ্তাহের ফাঁদ

**১. `cd /lab` লিখে ফেলা, যখন বলতে চেয়েছো `cd lab`** — শুরুর `/` পুরো অর্থ বদলে দেয়: `/lab` = root-এর ভেতরের lab (নেই), `lab` = এই ঘরের ভেতরের lab। `No such file or directory` দেখলেই আগে চেক: `/` বেশি না কম?

**২. `mkdir`-এ `-p` ভুলে যাওয়া** — গভীর পথ বানাতে সবসময় `mkdir -p`।

**৩. `>` দিয়ে পুরনো ফাইল মুছে ফেলা** — `>` মুছে লেখে, `>>` যোগ করে। Production config-এ ভুল `>` = ক্লাসিক দুর্ঘটনা।

**৪. `find`-এর wildcard quote না করা** — উপরের 🧪 মনে করো।

**৫. `grep` দিয়ে ফাইলের নাম খোঁজা** — নাম = `find`-এর কাজ, ভেতরের লেখা = `grep`-এর। উল্টো করলে দুজনেই চুপ।

| কাজ                | কমান্ড                       | মনে রাখো                  |
| ------------------ | ---------------------------- | ------------------------- |
| আমি কোথায়?        | `pwd`                        | present working directory |
| এই ঘরে কী আছে?     | `ls`                         | list                      |
| ঘর বদলাও           | `cd <path>`                  | `/` দিয়ে শুরু = absolute |
| বাড়ি ফেরো         | `cd ~`                       | ~ = home                  |
| এক ধাপ উপরে        | `cd ..`                      | .. = parent               |
| গলিসহ ঘর বানাও     | `mkdir -p a/b/c`             | -p = parents-ও            |
| স্ক্রিনে ছাপো      | `echo "hi"`                  | console.log               |
| ফাইলে লেখো (মুছে!) | `echo "hi" > f.txt`          | `>>` = যোগ করো            |

| ভেতরে খোঁজো        | `grep -r "<লেখা>" <কোথায়>`  | চিঠির ভেতর                |
| জুড়ে দাও          | `cmd1 \| cmd2`               | মুখ → কান                 |

## 🎯 Key Takeaways

1. Linux-এ **সবকিছু একটাই উল্টো গাছ** — শুরু root `/` থেকে।
2. **শুরুতে `/` = absolute, নেই = relative** — আজকের এক নম্বর কনসেপ্ট।
3. `pwd` / `ls` / `cd` — কোথায় আছি / কী আছে / সরে যাও — ৮০% navigation এটুকুই।
4. `find` = নাম (DFS-এ গাছ হাঁটে), `grep` = ভেতরের লেখা, `locate` = পুরনো ইনডেক্স খাতা।
5. `>` মুছে লেখে, `>>` যোগ করে, `|` output-কে পরের কমান্ডের input বানায়।

## 🎤 Interview-এ যদি জিজ্ঞেস করে...

**Q: absolute আর relative path-এর পার্থক্য? script-এ কোনটা ব্যবহার করবে?**

> Absolute path root `/` থেকে শুরু, যেকোনো working directory থেকে একই জায়গায় resolve করে; relative path current directory-র সাপেক্ষে। Script/cron/CI pipeline-এ absolute নিরাপদ — script কোন directory থেকে চলবে নিশ্চয়তা নেই। Relative সুবিধাজনক interactive কাজে।

**Q: production সার্ভারে একটা error কোন ফাইল থেকে আসছে জানো না — কীভাবে খুঁজবে?**

> দুই স্তরে: `find /var/log -name "*.log" -type f` দিয়ে log ফাইল দেখব, তারপর `grep -r "error-এর অংশ" /var/log/` দিয়ে content match। দরকারে pipe: `grep -ri "timeout" /var/log/ | grep "nginx"`।
