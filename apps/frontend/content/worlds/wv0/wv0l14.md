এতদিনে তুমি তিনটা জিনিস চিনেছো, মনে করো:

- **Lab 1:** `ls -la` চালিয়ে দেখেছিলে অদ্ভুত কিছু লাইন — `-rw-r--r-- 1 root root 14 ... meeting.txt`। তখন বলেছিলাম _"প্রথম অক্ষরটুকু শুধু চেনো — বাকিটা পরের বস।"_
- **User Management ক্লাস:** instructor বোর্ডে এঁকেছিলেন — `📁 File System → 👥 Group → 👤 User`, আর বলেছিলেন _"একটা group-কে একটা node-এর access দাও, user-কে সেই group-এ ঢোকাও।"_
- **Lab 2:** shajal-minhaz-দের দল বানিয়ে minions আর devops-এ ঢুকিয়েছো।

কিন্তু একটা প্রশ্নের উত্তর এখনো ঝুলে আছে: **"access দাও" মানে ঠিক কী দাও? কোথায় লেখা থাকে যে webteam এই ফোল্ডারে লিখতে পারবে আর অন্যরা পারবে না?**

উত্তরটাই আজকের লেসন: সেই `rw-r--r--` — **৯টা ছোট্ট অক্ষর, যেগুলো পুরো Linux-এর দারোয়ান।** আগের লেসনের ভাষায়: বিল্ডিং চিনেছো (file system), চাবির মালিক চিনেছো (user/group) — আজ চিনবে **তালাটাকেই।** 🔐

## 🤔 WHY — এক লাইনের ভুলে কোম্পানি ডোবে

- ইন্টারনেটে হাজারটা টিউটোরিয়াল বলে _"কাজ না করলে `chmod 777` মেরে দাও"_ — এটা **সার্ভারের সব দরজা খুলে রাস্তায় চাবি ফেলে দেওয়া।** কেন, আজ বুঝবে
- `ssh` কখনো চালালে দেখবে: private key-র permission ৬০০ না হলে ssh **কাজই করে না** — ইচ্ছা করে
- **Docker security** (instructor-এর সেই কথা): container-এর user-কে নিজের ফোল্ডারে আটকে রাখা হয় ঠিক এই permission দিয়েই — `chown appuser:appgroup /app`
- আর Lab 1-এর troubleshooting টেবিলের প্রথম error-টা মনে আছে? **Permission Denied** — আজকের পর ওটা আর রহস্য না

## 🧭 CONCEPT JOURNEY

### Step 1: ৯টা অক্ষর পড়তে শেখো — তিন জোড়া তিনটা প্রশ্ন

Lab 1-এর সেই লাইনটা আবার দেখো, এবার পুরোটা ভাঙি:

```
-  rw-  r--  r--   1  root  root  14  Jan 26 20:42  meeting.txt
↑   ↑    ↑    ↑        ↑     ↑
│   │    │    │      owner  group
│   │    │    └─ Others  — বাকি সবাই কী পারবে
│   │    └────── Group   — মালিক-দলের সদস্যরা কী পারবে
│   └─────────── Owner   — মালিক নিজে কী পারবে
└─────────────── টাইপ (- file, d directory) ← এটা আগেই চিনতে
```

প্রতিটা তিন-অক্ষরের জোড়ায় একই তিনটা প্রশ্ন:

| অক্ষর | মানে    | ফাইলে                      | directory-তে                         |
| ----- | ------- | -------------------------- | ------------------------------------ |
| `r`   | read    | পড়তে পারবে                | ভেতরের **তালিকা** দেখতে পারবে (`ls`) |
| `w`   | write   | বদলাতে পারবে               | ভেতরে **তৈরি/মুছতে** পারবে           |
| `x`   | execute | **চালাতে** পারবে (script!) | ভেতরে **ঢুকতে** পারবে (`cd`!)        |

> **Analogy:** ফাইল হলো একটা খাতা — `r` পড়া, `w` লেখা, `x` "খাতাটা recipe হলে রান্না করা।" আর directory হলো ঘর — `r` দরজার ফাঁক দিয়ে ভেতরের জিনিসের **লিস্ট দেখা**, `w` ঘরে জিনিস **রাখা/সরানো**, `x` ঘরে **পা রাখা।** শেষেরটাই সবচেয়ে অবাক করা: `x` ছাড়া `cd`-ই হয় না!

### Step 2: কার জন্য কোন জোড়া — আগের লেসনের সংযোগ

Linux যখন ঠিক করে তুমি একটা ফাইলে কী পারবে, সে **ঠিক একটা জোড়া** বেছে নেয় — ক্রমানুসারে প্রথম যেটা মেলে:

1. তুমি কি ফাইলটার **owner**? → Owner-জোড়া (`rw-`)
2. না হলে — তুমি কি ফাইলের **group-এর সদস্য** (primary বা supplementary — Lab 2-এর `id` মনে করো!)? → Group-জোড়া
3. কোনোটাই না? → **Others**-জোড়া

**এটাই instructor-এর সেই ছবির সম্পূর্ণ রূপ:**

```
📁 File/Directory ──(owner, group লেখা আছে গায়ে)──→ 👥 Group ──(সদস্যপদ)──→ 👤 User
       + ৯ অক্ষরের তালা (rwx rwx rwx)
```

User পরোক্ষভাবে ফাইলের permission পায় **group-এর মাধ্যমে** — এই এক বাক্যেই পুরো মডিউলের সারমর্ম।

### Step 3: অক্ষর থেকে সংখ্যা — ৪-২-১-এর নামতা

প্রতিটা permission-এর একটা দাম আছে: **r = 4, w = 2, x = 1।** এক জোড়ার দাম = যোগফল। তিন জোড়া = তিন অঙ্কের সংখ্যা:

```
rw-  =  4+2+0  =  6      rwx  =  4+2+1  =  7
r--  =  4+0+0  =  4      r-x  =  4+0+1  =  5
-w-  =  0+2+0  =  2      --x  =  0+0+1  =  1
```

তাহলে `-rw-r--r--` = **644**, আর `-rwxr-xr-x` = **755।** ব্যস — Lab 1-এর সেই রহস্যময় লাইন এখন তোমার কাছে খোলা বই।

মুখস্থ না করে **চারটা প্যাটার্ন** চেনো — বাস্তবের ৯০% এই চারটাই:

| Octal   | দেখতে       | কোথায়                    | কেন                                |
| ------- | ----------- | ------------------------- | ---------------------------------- |
| **644** | `rw-r--r--` | সাধারণ ফাইল, config, HTML | মালিক লেখে, সবাই পড়ে              |
| **755** | `rwxr-xr-x` | script, directory         | মালিক সব, বাকিরা পড়ে+চালায়/ঢোকে  |
| **600** | `rw-------` | SSH private key, secret   | **শুধুই মালিক** — আর কেউ কিছুই না  |
| **775** | `rwxrwxr-x` | team-shared directory     | দল মিলে লেখে, বাইরের লোক শুধু দেখে |

### Step 4: `chmod` — তালা বদলাও (দুই ভঙ্গিতে)

**ভঙ্গি ১ — সংখ্যায় (octal):** পুরো তালাটা একবারে বসাও:

```bash
chmod 644 index.html      # rw-r--r--
chmod 755 deploy.sh       # rwxr-xr-x
chmod 600 secret.conf     # rw------- — গোপন জিনিসের নিয়ম
chmod -R 775 /var/www/mysite   # -R = ভেতরের সবকিছুতেও
```

**ভঙ্গি ২ — কথায় (symbolic):** শুধু যেটুকু বদলাতে চাও:

```bash
chmod u+x deploy.sh       # u=owner-কে execute দাও (script চালু!)
chmod g-w plan.txt        # g=group-এর কাছ থেকে write কেড়ে নাও
chmod o-r secret.conf     # o=others-এর read বন্ধ
chmod a+x runme.sh        # a=all — সবাইকে execute
```

> মনে রাখার সূত্র: **u**ser(owner)/**g**roup/**o**thers/**a**ll + `+`(দাও) `-`(কাড়ো) `=`(ঠিক এটাই বসাও) + rwx। Octal = পুরো তালা বদল, symbolic = একটা চাবি যোগ/বিয়োগ।

### Step 5: `chown` আর `chgrp` — মালিকানা বদলাও

তালা বদলানো এক জিনিস, **মালিক বদলানো** আরেক:

```bash
chown shajal report.txt              # owner বদলাও
chown root:webteam /var/www/mysite   # owner ও group — একসাথে (কোলন!)
chown -R root:webteam /var/www/mysite  # ভেতরের সবকিছুসহ
chgrp devops plan.txt                # শুধু group বদলাতে
```

⚠️ Lab 2-এর সংযোগ: `chown`-এ যে user/group-এর নাম দেবে সেটা **আগে থেকে থাকতে হবে** — নাহলে `invalid user`। দল আগে (`groupadd`), মালিকানা পরে।

### Step 6: সব জোড়া লাগাও — webteam-এর গল্প

এখন পুরো সিস্টেমটা এক scenario-তে (এটাই পরের ল্যাবের preview): shajal আর minhaz মিলে একটা website-এ কাজ করবে। দুজনেই যেন লিখতে পারে, কিন্তু **বাইরের কেউ যেন শুধু দেখতে পারে:**

```bash
groupadd webteam                      # ১. দল           (Lab 2-এর স্কিল)
usermod -aG webteam shajal            # ২. সদস্য         (Lab 2-এর স্কিল)
usermod -aG webteam minhaz
mkdir -p /var/www/mysite              # ৩. ঘর           (Lab 1-এর স্কিল)
chown root:webteam /var/www/mysite    # ৪. ঘরের গায়ে দলের নাম   (আজ!)
chmod 775 /var/www/mysite             # ৫. তালা: দল=rwx, others=r-x (আজ!)
```

এখন Linux-এর চোখে: shajal `/var/www/mysite`-এ ফাইল বানাতে চাইলো → সে owner (root)? না → সে কি webteam-এর সদস্য? **হ্যাঁ** (supplementary!) → group-জোড়া `rwx` → **অনুমতি।** আর অচেনা কেউ? others-জোড়া `r-x` → দেখতে পারবে, লিখতে পারবে না।

**তিন লেসনের তিনটা স্কিল এক জায়গায়** — এটাই DevOps-এ প্রতিদিনের setup।

### Step 7: বোনাস চিহ্নগুলো — চিনে রাখো (গভীরে পরে)

`ls -l`-এ মাঝেমধ্যে অদ্ভুত অক্ষর দেখবে — এগুলোর নাম শুনে রাখো:

- `-rwsr-xr-x` — **s** = SUID: চালালে owner-এর ক্ষমতায় চলে (যেমন `/usr/bin/passwd` — সাধারণ user-ও নিজের password বদলাতে পারে, কারণ passwd প্রোগ্রামটা root-এর ক্ষমতায় চলে)
- `drwxrwxrwt` — **t** = sticky bit: `/tmp`-এ থাকে — সবাই লিখতে পারে, কিন্তু **মোছা যায় শুধু নিজেরটা**
- `umask` — নতুন ফাইল জন্মের সময় default permission কে ঠিক করে (সাধারণত 022 → ফাইল 644, dir 755)

## 🚫 COMMON MISTAKES — production-এর পাঁচ পাপ

**১. `chmod 777` — সর্বজনীন মাস্টার-চাবি।** "কাজ করছে না" সমস্যার আসল সমাধান হলো _কার_ permission লাগবে বের করা (owner? group?) — সবাইকে সব দিয়ে দেওয়া না। ৭৭৭ মানে যেকোনো user (হ্যাক-হওয়া app-এর user-সহ) তোমার ফাইল বদলাতে পারে।

**২. SSH key ৬০০ না রাখা** — `WARNING: UNPROTECTED PRIVATE KEY FILE!` বলে ssh সংযোগই প্রত্যাখ্যান করবে। গোপন জিনিস = 600, ব্যতিক্রম নেই।

**৩. Directory থেকে `x` কেড়ে নেওয়া** — `chmod 644 somedir` করলে কেউ ঢুকতেই পারবে না, নিজেও না। Directory-র জোড়ায় r-এর সাথে x প্রায় সবসময় জোড়া বাঁধা।

**৪. `-R` ভুলে যাওয়া (বা ভুল জায়গায় দেওয়া)** — `chown root:webteam /var/www/mysite` শুধু ফোল্ডারটার গায়ে নাম লেখে, **ভেতরের ফাইলে না।** পুরনো ফাইলসহ দিতে `-R`। উল্টোদিকে `chmod -R 777 /` টাইপ করা মানে সার্ভারের মৃত্যু।

**৫. group permission দিয়ে ভুলে যাওয়া user-টা group-এই নেই** — `chmod 775` + `chown :webteam` সব ঠিক, তবু shajal লিখতে পারছে না? `id shajal` চালাও — সে কি আদৌ webteam-এ? (Lab 2-এর `-aG` মনে করো!)

## 📋 আজকের cheat-table

| কাজ             | কমান্ড                              | মনে রাখো                    |
| --------------- | ----------------------------------- | --------------------------- |
| তালা পড়ো       | `ls -l`                             | ৩ জোড়া: owner/group/others |
| সংখ্যার নামতা   | r=4 w=2 x=1                         | 644, 755, 600, 775          |
| পুরো তালা বসাও  | `chmod 755 <path>`                  | octal                       |
| চাবি যোগ/বিয়োগ | `chmod u+x` / `g-w` / `o-r` / `a+x` | symbolic                    |
| ভেতরসহ          | `chmod -R` / `chown -R`             | recursive                   |
| মালিক বদলাও     | `chown <user> <path>`               |                             |
| মালিক+দল        | `chown <user>:<group> <path>`       | কোলন                        |
| শুধু দল         | `chgrp <group> <path>`              |                             |
| dir-এ ঢোকা      | `x` লাগবেই                          | r = লিস্ট, w = তৈরি/মোছা    |
| গোপন ফাইল       | `600`                               | SSH key-র নিয়ম             |

## 🎯 Key Takeaways

1. **৯ অক্ষর = ৩ জোড়া (owner/group/others) × ৩ প্রশ্ন (r/w/x)** — Linux প্রথম যে জোড়া মেলে সেটাই প্রয়োগ করে।
2. **r=4, w=2, x=1** — 644 (ফাইল), 755 (script/dir), 600 (গোপন), 775 (team dir) — এই চারটাই বাস্তবের ভাষা।
3. Directory-তে **x = ঢোকা**, r = লিস্ট, w = ভেতরে তৈরি/মোছা — x ছাড়া cd-ই হয় না।
4. `chmod` তালা বদলায় (octal/symbolic), `chown user:group` মালিকানা — আর user/group **আগে existence থাকতে হয়।**
5. পুরো ছবি: **file-এর গায়ে group লেখা → group-এ user → user পরোক্ষে permission পায়** — `groupadd` → `usermod -aG` → `chown :group` → `chmod 775` — এই চেইনটাই DevOps-এর রোজকার setup।

## 🎤 Interview-এ যদি জিজ্ঞেস করে...

**Q: 644 আর 755-র পার্থক্য কী, কখন কোনটা?**

> 644 = rw-r--r-- (owner লেখে, সবাই পড়ে) — data/config/HTML ফাইলের জন্য। 755 = rwxr-xr-x — execute bit যোগ হয়, তাই script আর directory-র জন্য (directory-তে x মানে ঢোকার অনুমতি)। ফাইলে অকারণ x দেওয়া বদভ্যাস — যেটা চালানোর জিনিস না সেটা চালানোযোগ্য রাখা attack surface বাড়ায়।

**Q: একটা দলের সবাই একটা ফোল্ডারে লিখবে, বাইরের কেউ পড়তেও পারবে না — design করো।**

> `groupadd team` → সদস্যদের `usermod -aG team` → `chown -R root:team /folder` → `chmod -R 770 /folder`। 7(owner)7(group)0(others) — others-এর জোড়া সম্পূর্ণ ফাঁকা, তাই বাইরের কারো list/ঢোকা/পড়া কিছুই নেই। নতুন ফাইলেও group ঠিক রাখতে চাইলে directory-তে SGID (`chmod g+s`) বসানো যায়।
